# 10. 보안/거버넌스 고려

## 1. 개요
금융 텍스트를 다루는 감성 분석 API 모델이기 때문에 보안과 데이터 거버넌스는 필수입니다. 하지만 별도 보안팀이 없는 5명 규모의 개발실이므로, 최소한의 노력으로 최대한의 보안 효과를 내는 방향으로 Docker와 Kubernetes 옵션들을 세팅했습니다.

## 2. 권한의 최소화 (Least Privilege)

**2.1 Docker 컨테이너 레벨 보안**
- 기본적으로 도커를 띄우면 내부 프로세스가 `root` 권한을 갖습니다. 하지만 해커에게 컨테이너가 장악당하면 호스트 서버까지 위험해지므로, `Dockerfile` 내부에 별도 `mluser` 계정을 만들고 `USER mluser` 옵션을 줘서 **비루트(Non-root) 권한**으로 실행되게 만들었습니다. 
- 이로 인해 컨테이너를 탈취당하더라도 파일 시스템을 마음대로 지우거나 다른 프로세스를 건드리지 못합니다.

**2.2 K8s 보안 컨텍스트 설정**
- 매니페스트 배포 시, 앱 컨테이너 바깥 네트워크 대역이나 리눅스 커널에 접근하지 못하도록 `securityContext` 제약을 추가할 예정입니다(readOnlyRootFilesystem 활용 등).

## 3. 네트워크 및 인프라 보안

- **불필요한 포트 차단**: 
  - `Dockerfile`에서는 모델 서빙을 위한 `EXPOSE 8000` 번 포트 딱 하나만 열어뒀습니다. 
  - K8s 매니페스트 상에서도 외부 통신을 위한 로드밸런서가 연결될 때 허용되는 트래픽 이외에는 기본적으로 차단합니다.
- **이미지 사이즈 최소화**:
  - `python:3.10-slim` 베이스 이미지와 빌드 캐시를 써서 런타임에 필요한 패키지만 남겼습니다. "앱이 가벼울수록 해커가 이용할 공격 벡터(취약점)도 줄어든다"는 보안 대원칙을 따랐습니다.

## 4. 데이터(로그) 거버넌스

- 로깅을 구현하면서 가장 신경 써야 하는 부분은 수집된 텍스트 요청(Request Body) 안에 **고객의 PII(개인 식별 정보)**나 **IP 주소**, **재무 데이터**가 함께 적재되는 것을 막는 것입니다.
- 현재는 텍스트가 암호화되지 않고 들어오지만, 향후 도입 시 FastAPI 미들웨어 단이나 애플리케이션 프레임워크 안에 PII 마스킹 처리 로직을 넣을 계획입니다. (예: 정규식 기반 이메일, 주민번호 등 필터링) 
- 이 과정을 통과한 "안전한 데이터"만이 json-logger를 거쳐 Loki 스토리지로 보관됩니다.
