# 12. 전체 시스템 아키텍처 설계도
제약 사항(외부 SaaS 사용 금지, 인력 부재, K8s + Docker 필수)을 반영하여 "Zero-Ops와 완전 자동화"에 초점을 맞춘 전체 MLOps 및 서빙 아키텍처입니다.

## 아키텍처 다이어그램 (Mermaid)

아래 코드를 복사하여 [Mermaid Live Editor](https://mermaid.live/) 에 붙여넣으면 깔끔한 시스템 구성도로 시각화할 수 있습니다. (Draw.io 등에서도 바로 붙여넣어 생성 가능합니다.)

```mermaid
graph TD
    %% 사용자 및 트래픽 유입
    Client([User/Client]) --> |"HTTP POST /predict"| LB[LoadBalancer / Ingress]
    
    %% K8s 클러스터 내부 (서빙 영역)
    subgraph K8s Cluster [Kubernetes Cluster]
        LB --> Svc[ml-api-service<br/>ClusterIP]
        
        %% HPA & Deployment
        HPA((HPA)) -.->|"Scale Out (CPU > 70%)"| Deploy{Deployment}
        
        subgraph Deployment [ml-api-deployment]
            Svc --> Pod1(API Pod 1<br/>FastAPI + PyTorch)
            Svc --> Pod2(API Pod 2<br/>FastAPI + PyTorch)
            Pod1 -.-> Model[(KR-FinBert-SC)]
            Pod2 -.-> Model
        end
    end
    
    %% 관측성(Observability) 영역
    subgraph Observability [Monitoring & Logging 스택]
        Svc -.-> |"GET /health, /metrics"| Prom[Prometheus]
        Prom --> Grafana[Grafana Dashboard]
        
        Pod1 -.-> |JSON stdout| Promtail[Promtail Agent]
        Pod2 -.-> |JSON stdout| Promtail
        Promtail --> Loki[Loki Storage]
        Loki --> Grafana
    end
    
    %% CI/CD 영역
    subgraph CI/CD Pipeline [GitHub Actions]
        Dev([Developer]) --> |"Git Push (main)"| GH[GitHub Repo]
        GH --> Test(Pytest / Health Check)
        Test --> Build(Docker Buildx)
        Build --> Registry[Container Registry]
        Registry -.-> |"Image Pull"| Deploy
    end

    %% 알림
    Prom -.-> |"상태 이상 (5xx Error)"| Alert[AlertManager]
    Alert -.-> |Slack| Dev

    %% 스타일링
    classDef k8s fill:#326ce5,stroke:#fff,stroke-width:2px,color:#fff;
    classDef obs fill:#e6522c,stroke:#fff,stroke-width:2px,color:#fff;
    classDef cicd fill:#2088ff,stroke:#fff,stroke-width:2px,color:#fff;
    
    class Svc,Deploy,Pod1,Pod2,HPA k8s;
    class Prom,Grafana,Loki,Promtail obs;
    class GH,Test,Build,Registry cicd;
```

### 흐름 부연 설명
1. **CI/CD 파이프라인**: 5명의 개발자가 코드를 `main`에 올리면 깃허브 액션이 `pytest`를 돌리고 통과하면 도커 이미지를 말아서 배포 시킵니다.
2. **트래픽 서빙**: 유저의 추론 요청이 K8s 클러스터의 `ml-api-service`를 타고 들어가고, HPA의 감시 하에 CPU 부하가 높아지면 스케일아웃이 일어납니다.
3. **통합 관측성**: 메트릭(Prometheus)과 텍스트 로그(Promtail -> Loki) 데이터가 전부 Grafana 하나로 모이도록 설계했습니다. 장애 시 대시보드만 띄워두고 분석이 가능합니다.
